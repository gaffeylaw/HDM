2.1.7. 弹性域结构

  • 弹性域，弹性域结构，弹性域段关系介绍：

    弹性域下可以包含多个弹性域结构，每个弹性域结构由多个弹性域段组成，当要使用弹性域时，首先需要指定弹性域名称，再指定一个附加属性用来确定一个弹性域结构，弹性域结构最终确定了该弹性域的段值。

  • 弹性域段使

    用实例，以“产品附加信息”弹性域为例：

    1. 弹性域信息：

      Table 1. 

      弹性域编码弹性域名称
      PRODUCT_ADDITION_INFO产品附加信息


    2. 弹性域结构：

      Table 2. 

      弹性域结构编码结构名称结构描述
      COLOR_INFO颜色信息描述产品的颜色信息
      SHAPE_INFO形状信息描述产品的形状信息信息

  • 建立弹性域结构：

    1. 新建弹性域结构

      由于弹性域结构属于弹性域，因此在进入弹性域结构的操作之前，首先要选择一个弹性域，弹性域结构的关键属性：

      • 名称：弹性域结构的名称

      • 结构编码：弹性域结构的编码

      • 描述：弹性域结构的描述

      • 分隔符：由于弹性域结构由许多段组成，在很多情况下需要使用弹性域结构下所有段连接后的字符串，该分割符就是区分该链接字符串中的每个段。例如：有三段分别为：

        Table 5. 

        segment1segment2segment3
        亚太区中国上海


        设置分割符为"."，则链接后的字符串为：亚太区.中国.上海

      • 可用：标示该弹性域是否有效。

    2. 为弹性域结构添加弹性域段

      弹性域段是弹性域的最底层设计，多个弹性域段组成一个弹性域结构，当一条记录的弹性域结构定了，在该记录对应的记录中，对的segment段就存放了该弹性域结构下弹性域段的信息。

      以表2和表3为例，如果选择结构为COLOR_INFO则表中的segment存储为产品 的颜色信息，存储形式如 下：

      Table 6. 

      segment1segment2segment3
      红色黑色蓝色


      如果选择结构为SHAPE_INFO则表中的segment存储的为产品的形状信息，存储 形式如下：

      Table 7. 

      sement1segment4segment5
      120cm80cm70cm


      弹性域段的关键属性：

      • 段编码：弹性域段对应的数据库存储字段，在设计弹性域时，需要为弹性域对应的表设计弹性域字段，通常为segment1..segment15，15个段。段编码为为下拉列表，同一弹性域下的段编码不能重复；

      • 段名称：当前段存储信息的名称，在使用弹性域的界面中，该字段值是显示在界面中来标示当前域所代表的意义；

      • 描述：当前弹性域段的详细描述；

      • 值集：该弹性域段对应的一个值列表，该值集必须在值集定义中设置；

      • 显示顺序：弹性域段在界面上的显示顺序；

      • 可用：该弹性域段是否有效。
