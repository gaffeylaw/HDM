事故单-编辑

  1. 界面所需角色：

    • 提单人员

    • 服务人员

  2. 界面操作：

    该界面对应操作是输入创建或编辑事故单所需要的信息，其中带有红色标记的为必输项，其它的为可选项，字段解释如下：

    Table 3. 

    字段解释备注
    服务  
    标题事故单的标题该标题应该能简要概括事故单的意义
    信息事故单的具体内容建议事故单的内容包括：系统，界面，操作步骤，错误信息
    请求人事故单的请求人 
    联系人(电话)事故单的联系人电话 
    紧急度事故的紧急度该故障需要修复的紧急程度
    影响度事故的影响度该事故的影响范围，可以按部门，公司，全部公司来进行参考设置
    支持组该事故单对应的支持组如果不知道选谁，此处留空，系统会自动分配
    支持人员该事故单的支持人员如果不知道应该选谁，此处留空，系统会自动分配
    请求类型该值直接从服务中关联下拉列表的值来自快速编码 ICM_REQUEST_TYPE_CODE
    来源类型事故单来源类型下拉列表的值来自快速编码 ICM_REQUEST_TYPE_CODE
    状态该事故单所处的状态该状态可以由服务台人员和支持组人员进行修改。


  3. 关键点：

    • 紧急度的确定：紧急度是指当前事故对业务影响的紧急性，紧急度与事故的复杂程度不一定是成正比的，例如：打印机驱动错误不是非常复杂的问题，但是由于销售人员急需要打印一份报价单给客户，那么此时该问题的紧急度应该是最高的；

    • 影响度的确定：影响度与紧急度共同确定事故单的优先级，影响度是指当前事故对业务的影响范围，一般情况下是按照公司架构的影响范围进行确定，例如按事故影响了部门业务，分公司业务，整个区域的业务，分为低，中，高；

  4. 界面导航：

    输入各个数据项后，可以点击“保存”按钮保存当前事故单，电影“取消”按钮取消当前事故单；
